1. CloudTrail

Field Detail

Severity 🔴 CRITICAL
Issue No CloudTrail trails were found in this AWS account

Description: AWS CloudTrail is the primary mechanism for recording API activity and management events across an AWS account. The complete absence of any CloudTrail trail means that no audit log exists for any action taken within this environment — whether by users, roles, or AWS services. This eliminates the ability to detect malicious activity, investigate incidents, demonstrate compliance, or perform forensic analysis after a security event.

Risk: - No visibility into who accessed or modified any resource - Inability to detect credential misuse, privilege escalation, or data exfiltration - Non-compliance with virtually all major security frameworks (PCI-DSS, ISO 27001, SOC 2, CIS Benchmarks) - Incident response rendered severely impaired

3. S3 — Bucket Misconfiguration

HIGH — Bucket: dev-project-assets-211125321299-eu-west-2-an

Field Detail
Resource dev-project-assets-211125321299-eu-west-2-an
Severity 🟠 HIGH
Issue Bucket encryption is not using AWS KMS

Description: The bucket is either unencrypted or encrypted using Amazon S3-managed keys (SSE-S3) rather than AWS Key Management Service (SSE-KMS). SSE-KMS provides additional controls including key rotation, access auditing via CloudTrail, and the ability to revoke access at the key level.

Recommendation: - Enable SSE-KMS encryption on this bucket using a Customer Managed Key (CMK). - Apply a bucket policy that denies s3:PutObject requests that do not include the x-amz-server-side-encryption: aws:kms header. - Enable automatic key rotation on the KMS CMK.

HIGH — Bucket: drone-footage-dev-211125321299-eu-west-2-an (2 Findings)

Field Detail
Resource drone-footage-dev-211125321299-eu-west-2-an
Severity 🟠 HIGH
Issues (1) Public Access Block not configured; (2) Bucket encryption not using AWS KMS

Description: This bucket has two distinct misconfigurations:

  1. Public Access Block Not Configured: All four S3 Public Access Block settings (BlockPublicAcls, IgnorePublicAcls, BlockPublicPolicy, RestrictPublicBuckets) are not enabled. This means the bucket could be made public — either intentionally or accidentally — through ACLs or bucket policies. Given this bucket contains drone footage (potentially sensitive operational or personally identifiable imagery), this is of particular concern.

  2. No KMS Encryption: As with the asset bucket above, the absence of KMS encryption reduces key management control and auditability.

Recommendation: - Immediately enable all four S3 Block Public Access settings at both the bucket and account level. - Audit the bucket's ACL and bucket policy to ensure no public access grants exist. - Enable SSE-KMS encryption with a dedicated CMK for this bucket. - Consider enabling S3 Object Lock if the footage has compliance or evidential retention requirements. - Enable S3 Access Logging to track who is accessing the drone footage.

Key Recommendations Summary

  1. Enable CloudTrail immediately — The absence of audit logging is the single most critical gap and must be resolved before any meaningful incident detection or response is possible.
  2. Adopt a least-privilege IAM strategy — Conduct a full IAM access review using AWS IAM Access Analyzer and remove all unused permissions. Use permission boundaries and SCPs where applicable.
  3. Eliminate all public internet exposure on sensitive ports — No management ports (22, 3389), data store ports (9200, 5601, 5044), or catch-all rules should be open to 0.0.0.0/0.
  4. Enforce KMS encryption across all S3 buckets — Implement a Service Control Policy (SCP) to prevent creation of unencrypted S3 buckets.
  5. Implement infrastructure security standards — Adopt AWS Security Hub with CIS AWS Foundations Benchmark and AWS Foundational Security Best Practices standards to continuously monitor for regressions.
  6. Segregate development and production environments — The presence of numerous launch-wizard-* security groups suggests ad-hoc instance creation. Enforce infrastructure-as-code (e.g., Terraform with security guardrails) and remove manually created resources.
